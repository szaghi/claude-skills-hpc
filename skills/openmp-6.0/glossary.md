# Glossary — OpenMP 6.0

Key terms from the spec Glossary (Ch2) + directive/clause vocabulary. Format: **Term** — definition (Ch).

**allocator** — object that allocates from a memory space with tunable traits (Ch8).
**atomic** — construct giving lock-free read/write/update/capture/compare of one location (Ch16).
**barrier** — point where all team threads rendezvous; implies a flush (Ch16).
**binding region / binding thread set** — the region/threads a construct's effect is scoped to (Ch2).
**canonical loop nest** — the loop form (integer/iterator counter, invariant bounds, computable trip count) required for parallelization (Ch6).
**construct** — an executable directive + its associated code (Ch2).
**contention group** — tasks run by one thread pool; threads don't migrate between groups (Ch1).
**critical** — region mutual-exclusion construct, optionally named (Ch16).
**data environment** — the variables and their attributes available to a region/task (Ch7).
**data-mapping attribute** — `to`/`from`/`tofrom`/`alloc`: host↔device movement via `map` (Ch7).
**data-sharing attribute** — `shared`/`private`/`firstprivate`/`lastprivate`/`reduction`/`linear` (Ch7).
**declare target** — mark a function/variable for device residence/compilation (Ch15).
**declare variant** — register a context-selected alternate function implementation (Ch9).
**defaultmap** — implicit map behavior on `target`; `private` behavior = data-sharing private (errata) (Ch7).
**depend** — clause building task data-flow dependences (in/out/inout/mutexinoutset) (Ch14, 16).
**device data environment** — storage a device sees, populated by `map`/`target data` (Ch15).
**dispatch** — construct controlling variant substitution at a call site (Ch9).
**distribute** — worksharing across the teams of a league (Ch13).
**explicit task** — task generated by `task`/`taskloop` (deferrable, may be stolen) (Ch14).
**flush** — memory operation establishing a consistent view; acquire/release build happens-before (Ch16).
**free-agent thread** — 6.0 thread not bound to a team, available to execute tasks (Ch3).
**gang/worker/vector** — (OpenACC term) — OpenMP analog: teams/threads/SIMD lanes (Ch1, 12).
**happens-before** — ordering established by a release flush before an acquire flush of the same data (Ch16).
**ICV** — internal control variable; scopes global/device/implicit-task/data-environment (Ch3).
**implicit task** — one per team thread, tied (only its thread runs it) (Ch1, 2).
**interop** — construct retrieving foreign-runtime handles (e.g. CUDA stream) (Ch15).
**league** — the set of teams created by `teams` (Ch1, 12).
**lastprivate** — private, with the last-iteration/section value copied out (Ch7).
**loop construct** — descriptive concurrent worksharing; implementation chooses mapping (Ch13).
**map** — clause moving list items into/out of a device data environment (Ch7, 15).
**masked** — construct restricting execution to filtered thread(s); replaces `master` (Ch12).
**memory space** — storage resource (default/large/const/high-bw/low-lat) (Ch8).
**metadirective** — choose among candidate directives by OpenMP context (Ch9).
**nowait** — drop the implicit end barrier of a worksharing/target region (Ch13).
**OMPD** — third-party, out-of-process tool (debugger) interface (Ch18).
**OMPT** — first-party, in-process tool (profiler) interface (Ch18).
**ordered** — enforce source order / loop-carried (doacross) dependences in a loop (Ch16).
**parallel** — construct forming a team of threads with an end barrier (Ch12).
**primary thread** — the thread that encountered `parallel`; thread 0 of the team (Ch1, 12).
**private / firstprivate / shared** — per-thread uninit / per-thread init-from-original / one shared instance (Ch7).
**proc_bind** — thread→place affinity policy (primary/close/spread) (Ch4, 12).
**reduction** — combine per-thread/team partials by an operator (Ch7).
**region** — all code executed during a construct instance, incl. callees (dynamic) (Ch1, 2).
**replayable** — task/taskloop clause for taskgraph record/replay; default property = unique (errata) (Ch14).
**requires** — directive mandating implementation features (USM, reverse_offload, …) (Ch10).
**safesync** — 6.0 `parallel` clause asserting device sync safety (Ch12).
**schedule** — worksharing-loop iteration distribution (static/dynamic/guided/auto/runtime) (Ch13).
**sections / single** — independent blocks one-thread-each / one-thread-only-then-barrier (Ch13).
**simd** — execute loop iterations across SIMD lanes (Ch12).
**target** — construct offloading a region to a device (Ch15).
**taskgraph** — 6.0 construct recording/replaying a task graph (Ch14).
**taskgroup / taskwait** — wait for descendant / child tasks (Ch14, 16).
**team** — the threads executing a `parallel` region (Ch1, 12).
**teams** — construct creating a league of teams (device offload) (Ch12, 15).
**threadprivate** — per-thread persistent copy of a global/static variable (Ch7).
**unified_shared_memory** — `requires` feature collapsing host/device address space (Ch10).
