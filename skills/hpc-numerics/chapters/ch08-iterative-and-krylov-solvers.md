# Chapter 8: Iterative & Krylov Solvers

## Core Idea
For large sparse systems where direct LU fill-in is catastrophic, **iterative methods** solve `Ax = b` by repeatedly improving a guess — using only **matrix-vector products** (which preserve sparsity) and never forming the factorization. The cost is that **convergence rate depends on conditioning**, so **preconditioning** (improving the effective condition number) is the decisive performance lever.

## Frameworks Introduced

- **Stationary iterative methods** (the classics): split `A = M − N` and iterate `x_{k+1} = M⁻¹(N x_k + b)`.
  - **Jacobi** (M = diagonal of A), **Gauss-Seidel** (M = lower triangle), **SOR** (over-relaxed Gauss-Seidel).
  - Convergence is governed by the **spectral radius** of the iteration matrix `M⁻¹N` (< 1 to converge); slow for ill-conditioned problems. Mostly superseded by Krylov methods but conceptually foundational and useful as smoothers/preconditioners.

- **Krylov subspace methods** (the modern workhorses): build the solution in the **Krylov subspace** `span{b, Ab, A²b, …}` generated by repeated matrix-vector products.
  - **Conjugate Gradient (CG)** — for **symmetric positive definite** matrices; minimizes error in the A-norm, converges in ≤ n steps exactly but far fewer in practice. The method of choice for SPD systems (e.g. Poisson).
  - **GMRES** — for general (nonsymmetric) matrices; minimizes the residual but stores a growing basis (often **restarted**, GMRES(m)).
  - **BiCGSTAB**, **MINRES**, etc. for other matrix classes.

- **Preconditioning** (the key to fast convergence): replace `Ax = b` with `M⁻¹Ax = M⁻¹b` where `M ≈ A` but is cheap to invert. A good preconditioner makes `M⁻¹A` well-conditioned (eigenvalues clustered near 1), slashing iteration count.
  - Common: **Jacobi/diagonal**, **incomplete LU (ILU)** (sparse approximate factorization with limited fill-in), **multigrid** (optimal for elliptic PDEs — O(n) work), domain-decomposition preconditioners.

## Key Concepts
- **Convergence ∝ conditioning**: the iteration count for CG scales with `√κ(A)`; an ill-conditioned system converges slowly or not at all. **The preconditioner choice dominates performance** — far more than the Krylov method itself.
- **Matrix-free operation**: iterative methods only need `A·v` (a sparse matvec), never `A` explicitly — so they preserve sparsity and can even work without storing the matrix (apply the stencil directly).
- **Residual vs error**: you can measure the **residual** `r = b − Ax` (computable) but not the true error (needs the answer); convergence is judged by `‖r‖` falling below a tolerance. Beware: a small residual on an ill-conditioned system can still mean a large error.
- **Multigrid as the gold standard for elliptic PDEs**: by solving on a hierarchy of grids (smooth high-frequency error on fine grids, low-frequency on coarse), multigrid achieves O(n) work — optimal — where unpreconditioned CG would scale far worse.

## Mental Models
- **Sparse + large ⇒ iterative, not direct** — when direct LU fill-in would be catastrophic (Ch 7), iterate using only matvecs; you trade exactness for sparsity preservation.
- **The preconditioner is where the performance is** — a Krylov method on a raw ill-conditioned system crawls; with a good preconditioner (ILU, multigrid) it flies. Spend your effort here, not on the Krylov variant.
- **CG for SPD, GMRES for general** — match the method to the matrix structure; using GMRES on an SPD system wastes CG's superior properties, and CG on a nonsymmetric matrix is invalid.
- **Use multigrid for elliptic PDEs** — it's the asymptotically optimal solver/preconditioner for Poisson-like problems; nothing else achieves O(n).
- **Judge convergence by the residual, but watch conditioning** — a small residual on an ill-conditioned problem doesn't guarantee a small error (Ch 4).

## Key Takeaways
1. Iterative methods solve large sparse systems using only matrix-vector products — no factorization, no fill-in (the answer to Ch 7's fill-in problem).
2. Krylov methods (CG for SPD, GMRES for general) build the solution in `span{b, Ab, A²b, …}`; match the method to the matrix class.
3. Convergence scales with conditioning (CG ∝ √κ(A)) — so **preconditioning is the dominant performance lever**, more than the Krylov method itself.
4. Preconditioners: Jacobi (cheap/weak) → ILU (general) → **multigrid** (optimal O(n) for elliptic PDEs).
5. Judge convergence by the residual `‖b − Ax‖`, but remember a small residual on an ill-conditioned system can still hide a large error.
